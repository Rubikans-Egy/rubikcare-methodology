RUBIKCARE PSP · DETAILED SPECIFICATIONS

Patient Support Program Platform

End-to-end digital infrastructure for designing, managing, and measuring patient support programs — from program design to real-world evidence.

1. Program Design & Configuration
Create and configure PSP programs with full control over eligibility, budget, and target audience.
💊
Medication Configuration
Define which products are included in the program — single or multiple medications per program.
  • Single product programs
  • Multi-product treatment bundles
  • Product substitution rules
🎯
Specialty Targeting
Target specific medical specialties or leave open for all physicians.
  • Single specialty targeting
  • Multi-specialty programs
  • Open enrollment for all specialties
💰
Budget & Financial Controls
Set program budget and define financial support rules.
  • Fixed program budget
  • Per‑patient budget limits
  • Real‑time budget tracking
🏷️
Discount & Subsidy Configuration
Define financial support per product, per program, or per patient segment.
  • Percentage discount (e.g., 10%)
  • Fixed amount per dispense (e.g., 50 EGP)
  • Tiered discounts based on volume
  • Free medication (100% subsidy)
2. Treatment Protocol & Dispensing Schedule
Define clinical parameters and dispensing rules for each program.
💊
Dosage & Frequency
Set medication dosage instructions and dosing frequency per product.
  • Daily / Weekly / Monthly dosing
  • Custom dosing schedules
  • Auto‑calculated refill dates
🔄
Dispensing Cycles
Control how many times a patient can dispense and when.
  • Maximum number of dispenses
  • Days between dispenses
  • Total therapy duration
  • Auto‑expiration after last dispense
📌 Example: A 6‑month treatment program with 4 dispenses (every 45 days), 20 capsules per dispense, with a 10% discount on each dispense. System automatically calculates eligibility dates and sends reminders.
3. Token & QR Code Dispensing System
Secure, token‑based dispensing with QR codes for physician referrals and pharmacy fulfillment.
👨‍⚕️📱
Physician QR Code
Generated by the physician when inviting a patient. Scan to enroll patient instantly.
  • Unique QR per invitation
  • Expires after enrollment
  • Track which physician invited which patient
💊🔐
Dispensing Token (OTP)
Time‑limited one‑time password generated for each dispense authorization.
  • 6‑digit numeric token
  • Expires after 48 hours
  • Single‑use only
  • QR code version available for scanning
4. Automated Medication Reminders
Intelligent reminder system triggered by the first dispensed dose — no manual configuration required.
⏰
Dose Reminders
Automatically scheduled based on dosage instructions defined in program design.
  • Push notifications
  • SMS fallback
  • Customizable reminder times
📦
Refill Reminders
Triggered automatically when a patient is eligible for next dispense.
  • 3 days before eligibility date
  • Day of eligibility
  • Overdue reminders
🔔
Missed Dose Alerts
Notify patient and optionally caregiver when a dose is missed.
  • Missed dose detection
  • Optional escalation to physician
  • Adherence scoring
5. Reports & Analytics
Real‑time operational and clinical insights from program activity.
📊
Program Performance
Enrollment, active patients, drop‑off rates
💊
Adherence Report
MPR, PDC, missed doses, refill compliance
💰
Budget Utilization
Spent vs allocated, cost per patient
🏥
Geographic Distribution
Patient map by governorate, pharmacy coverage
👨‍⚕️
Physician Performance
Patients enrolled per physician, referral rates
💊
Dispensing Activity
Token usage, pharmacy performance, product mix
📋
Real‑World Evidence (RWE)
Patient reported outcomes, side effects, effectiveness
📈
ROI Analysis
Program cost vs incremental revenue, ROMI calculation
📊 All reports exportable to: Excel (XLSX), CSV, PDF. Scheduled reports available for email delivery.
